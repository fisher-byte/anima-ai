## 关键实现说明

### 1. 语义记忆检索 ✅

**实现**: `canvasStore.ts` 中的 `getRelevantMemories` 方法。
**逻辑**: 
- 提取用户当前输入的核心关键词。
- 遍历 `conversations.jsonl`，根据匹配关键词数量进行评分。
- 返回评分最高的前 3 条记录，并注入 System Prompt。

### 2. 知识图谱连线 (Edges) ✅

**实现**: `Edge.tsx` 与 `Canvas.tsx`。
**视觉**:
- 使用 SVG `path` 绘制。
- 同一 Category 的节点通过 `Edge` 联结，形成星型拓扑（连接到该类别第一个节点）。
- 连线采用虚线 (`strokeDasharray`)，降低视觉干扰。

### 3. Apple-like 动效集成 ✅

**实现**: `NodeCard.tsx` 与 `AnswerModal.tsx`。
**亮点**:
- 使用 `framer-motion` 的 `AnimatePresence` 处理删除按钮的淡入淡出。
- 节点创建时具有 Spring (弹簧) 物理反馈。
- 呼吸灯效果展示 AI 正在同步状态。

## 代码质量建议

1. **性能优化**: 
   - 随着节点增多，`updateEdges` 在每次拖拽时都会全量重新计算。目前节点较少（<50）性能尚可，后续建议引入 `Web Worker` 或增量更新逻辑。
2. **记忆精度**:
   - 当前的关键词匹配较为基础。建议未来引入 `vector embeddings` 实现真正的语义检索。
3. **视觉调优**:
   - 连线目前的“星型”拓扑可能在节点密集时显得杂乱，建议后续改为“邻近连线”或“最小生成树”布局。
